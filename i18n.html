A base class for creating i18n handler implementations

handle(rctx)¶
Adds self to rctx.vals as language_handler and activates the default language (looks it up in cookies or uses default_language)
class insanities.utils.i18n.gettext_support(localepath, default_language=None, languages=None, domain='insanities', load_from_cookie=None)¶

Request handler addding support of i18n

default_language - language used by default, it is activated by this handler in handle() method. If not passed to the constructor, used first item from languages attribute. At least default_language or langages must be specified.

languages - languages (“en”, “ru”) or locales (“en_GB”, “ru_RU”) code. If provided, only given language can be activated. If there is no default_language provided, first language is default.

localepath - a path to locale directory containing .mo file.

domain - gettext domain of translation.

load_from_cookie - http cookie name. If provided, handle() method activates language from this cookie with fallback to default_cookie.

get_translation(language)¶

Returns a translation object.

Adds a fallback to language without locale, if a language is particular locale and has not translation.

Adds a fallback to the default language, if it’s different from the requested language.

language - language (“en”, “ru”) or locale (“en_GB”, “ru_RU”) code.

Integration with forms¶

The internationalization module is integrated with forms using FormEnvironment class’s gettext and ngettext methods. These methods can be redefined in subclasses to customize initialization.

By default, internationalization is used by Converter error method. This method accepts name of the error, looks up it’s message in converters’ error_%(name)s attribute and translates the message.
