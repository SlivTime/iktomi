Converters¶

Converter¶

class insanities.forms.convs.Converter([null][, multiple])¶

Base converter with chaining support extend this class in order to get custom converter.

Converting:

to_python() method takes value from form and converts it to python type

from_python() method takes value as python object and converts it to string or something else widget can display

Chaining:

Result of first converter is passed as input value to second. for example:

convs.Char(max_length=2)|convs.Int()

will be a convertor which first validates if string’s length is 2 or less and after that converts it to integer

Error messages redefinition:

Set error_<type> parameter to your own message template, for example:

convs.Char(min_length=5,
           error_min_length='At least %(min_length)s characters required')`
from_python(value)¶
custom converters should override this
multiple¶
Signs if converter is multiple or not. Multiple converters usually accept and return lists.
null¶
If null is True Converter represents empty value (‘’ or None) as None

Html¶

class insanities.forms.convs.Html(**kwargs)¶

Converter for flexible cleanup of HTML document fragments. A subclass of Char.

Uses utils.html.Sanitizer instance to sanitize input HTML.

Construtor collects from given kwargs all of Sanitizer options and passes them into Sanitizer’s constructor.

For list properties it is allowed to use add_%s() interface:

Html(add_allowed_elements=['span'], add_dom_callbacks=[myfunc])
